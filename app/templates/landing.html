Proxmox API 설정

https://로 시작하여 /api2/json로 끝나는 전체 URL을 입력하세요.
기본값: 5초. 네트워크가 느릴 경우 늘려주세요.
Proxmox API 토큰 ID를 입력하세요 (예: user@pam!token)
NFS 설정
NAS NFS 허용 IP에는 이 컨테이너를 실행 중인 도커 호스트(LXC 등)의 실제 LAN 주소를 등록하세요.
※ 도커 브리지(예: 172.17.x.x)나 컨테이너 내부 IP가 아니라, NAS에서 직접 접근 가능한 호스트/LXC의 LAN 주소입니다.
예시: 192.168.x.x:/mnt/gshare *(rw,sync,no_subtree_check)
주의: NFS 서버 설정 시 보안을 위해 특정 IP 주소나 서브넷으로 접근을 제한하는 것을 권장합니다.
NFS 공유를 마운트할 로컬 경로를 입력하세요.
NFS 서버의 공유 경로를 입력하세요. (예: 192.168.1.100:/volume1/gshare)

VM 설정

Android VM을 종료할 때 사용할 웹훅 URL을 입력하세요.

모니터링 설정

일반적으로 event 모드를 권장합니다. NAS에서 이벤트 전달이 어려운 환경에서는 polling 모드를 사용하세요.
event 모드에서 `/api/folder-event` 요청의 `X-GShare-Token` 값과 동일해야 합니다.
이 값 이하로 CPU 사용량이 떨어지면 VM을 종료합니다.
연속으로 이 횟수만큼 임계치 이하로 측정되면 VM을 종료합니다.

SMB 공유 설정

기본 SMB 포트는 445입니다. 포트 충돌이 있을 경우 변경하세요.
